1. Leading the Flutter Development Team for Natural Calm – A Pharmaceutical E-Commerce App Prototype at Precise Tech

Project Overview:

At Precise Tech, I led a dynamic and enthusiastic team of junior Flutter developers to create a prototype for Natural Calm, a pharmaceutical e-commerce app. The app’s main goal was to provide users with an easy, secure, and efficient way to purchase natural health products online. While my team worked on the Flutter front-end, another team at Precise Tech handled the backend development using .NET.

Detailed Description:
  • Role: Lead Flutter Developer
  • Responsibilities: I was responsible for overseeing the development of the app’s user interface, ensuring that it was responsive, visually appealing, and easy to navigate. As part of the project, I ensured seamless communication between our Flutter team and the .NET backend team to ensure that the app could integrate effectively with the backend system.
  • Technologies Used: Flutter, Firebase, REST APIs for backend communication, .NET (by the backend team).
  • Achievements: Our team developed key features such as product browsing, user authentication, cart management, and payment integration, delivering a functional prototype for the client. We also managed to integrate real-time order tracking and notifications for the users.
Key Challenges:

One of the challenges was making sure that both teams (Flutter and .NET) communicated effectively throughout the project. We ensured that both ends were in sync, addressing any API integration or backend-related issues promptly. This experience strengthened my communication and leadership skills, as I had to mediate between the two teams to maintain workflow consistency.

2. Leading the Flutter Development Team for Sowa Tool – A Hardware E-Commerce App Prototype at Precise Tech

Project Overview:

As the lead developer for the Sowa Tool project at Precise Tech, I guided a young team of Flutter developers in building a prototype for a hardware e-commerce app. The app aimed to provide a smooth and efficient platform for users to browse and purchase various hardware tools and accessories. Much like the Natural Calm project, the backend for this project was also handled by another team using .NET.

Detailed Description:
  • Role: Lead Flutter Developer
  • Responsibilities: I managed the development of the user interface, collaborating closely with both my Flutter team and the .NET team to ensure that the mobile application functioned as intended with backend APIs. I worked to create a seamless user experience, focusing on the visual design, filtering features, and ease of navigation.
  • Technologies Used: Flutter, Firebase, REST APIs, .NET (for the backend).

Key Challenges:

Ensuring that the communication between the two teams (front-end and back-end) was clear and efficient was challenging, especially when handling multiple API calls and ensuring data consistency across platforms. As the team lead, I played a key role in troubleshooting any issues and bridging the gap between the two teams to maintain a unified workflow.

3. Current Project: Custom Windows POS System – Python Django Backend & Flutter Frontend (Collaborating with Yahya)

Project Overview:

Currently, I am working on a custom-built Point of Sale (POS) system designed specifically for Windows. This system is being developed with a Python Django backend and Flutter for the front-end interface. I am working closely with my friend Yahya on this project, where I am primarily responsible for the backend logic, while Yahya handles the frontend development.

Detailed Description:
  • Role: Backend Developer (Python/Django) & Frontend Developer (Flutter)
  • Responsibilities: I focus on writing and optimizing the backend code using Django, ensuring that key features such as inventory management, sales tracking, and user authentication work seamlessly. I also collaborate with Yahya to implement the frontend with Flutter, which includes designing a simple and intuitive user interface for employees to process sales quickly and efficiently.
  • Technologies Used: Python, Django, Flutter, PostgreSQL for the database, and REST APIs for communication between the front-end and backend.
  • Achievements: The POS system will include real-time inventory management, order processing, and detailed sales reports. This project will also allow users to manage sales transactions, print receipts, and track stock levels. The system’s integration with a Windows environment ensures compatibility with common retail hardware (like barcode scanners and receipt printers).
Key Challenges:

Building a custom POS system required close collaboration and attention to detail, especially in the integration between the backend and frontend. The biggest challenge was ensuring that the system could handle real-time updates without any lag, which is critical for a POS system in a fast-paced retail environment. Optimizing performance for Windows-based devices and integrating with peripheral hardware (like receipt printers) also posed technical challenges, which we are addressing by developing custom solutions.
